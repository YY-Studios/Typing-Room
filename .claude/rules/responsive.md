# 모바일 대응 (반응형) 규칙

## 기본 원칙

- **모바일 퍼스트**: 기본 스타일은 모바일 기준, 큰 화면은 `sm:` `md:` `lg:`로 확장
- 가로 스크롤 절대 금지: `overflow-x: hidden` 또는 레이아웃 재검토

## 브레이크포인트 기준 (Tailwind)

| 접두사 | 기준    | 용도                    |
| ------ | ------- | ----------------------- |
| (없음) | 0px~    | 모바일 기본             |
| `sm:`  | 640px~  | 큰 모바일 / 작은 태블릿 |
| `md:`  | 768px~  | 태블릿                  |
| `lg:`  | 1024px~ | 데스크탑                |

## 터치 타겟

- 모든 버튼/링크: 최소 **44×44px** (애플 HIG / WCAG 기준)
- 너무 작은 경우: `min-h-[44px] min-w-[44px]` 또는 `p-` 패딩으로 확장

## 금지 패턴

- 고정 px 너비 (`w-[800px]`) — `max-w-` + `w-full` 조합으로 대체
- `hover:` 전용 인터랙션 — 터치 기기는 hover 없음, `active:` 병행
- 작은 폰트 (`text-[10px]` 이하) 모바일에서 단독 사용

## 텍스트

- 모바일 최소 폰트: `text-xs` (12px)
- 줄 길이: 모바일에서 한 줄이 너무 길지 않도록 `max-w-prose` 또는 `px-` 여백 확보

## 이미지 / 미디어

- `<img>`: 반드시 `w-full` 또는 `max-w-full` 적용
- 고정 크기 이미지: `object-cover` + 컨테이너로 제어

## 검사

- `/responsive` 커맨드로 현재 컴포넌트 반응형 이슈 감사
