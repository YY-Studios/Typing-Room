# 접근성 (A11y) 규칙

## 시맨틱 HTML

- 버튼은 `<button>`, 링크는 `<a href>` — div/span에 onClick 금지
- 제목 계층 순서 유지: `h1` → `h2` → `h3`
- 폼 요소에는 반드시 `<label>` 연결

## ARIA

- 아이콘만 있는 버튼: `aria-label` 필수 (예: `<button aria-label="닫기">`)
- 장식용 이미지: `alt=""` (빈 문자열)
- 의미 있는 이미지: 내용을 설명하는 `alt` 텍스트
- 동적 콘텐츠 변경 시: `aria-live` 사용 고려

## 키보드 네비게이션

- 모든 인터랙티브 요소는 키보드로 접근 가능해야 함
- 커스텀 드롭다운/모달: 포커스 트랩 구현
- 모달 닫힐 때: 트리거 버튼으로 포커스 복귀
- `tabIndex={-1}` — 프로그래매틱 포커스용으로만 사용

## 색상 대비

- 일반 텍스트: 최소 4.5:1 비율
- 큰 텍스트 (18px+ 또는 bold 14px+): 최소 3:1 비율
- 색상만으로 정보 전달 금지 — 아이콘이나 텍스트 병행

## 포커스 스타일

- `outline: none` / `focus:outline-none` 단독 사용 금지
- 커스텀 포커스 스타일 대체 필수: `focus-visible:ring-2` 등

## 자동 검사

- 파일 저장 시 `eslint-plugin-jsx-a11y`가 자동으로 경고 표시
- `/a11y` 커맨드로 현재 컴포넌트 전체 감사 가능
