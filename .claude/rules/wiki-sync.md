# 위키 동기화 규칙

## 언제 업데이트하나

작업 세션에서 아래 중 하나라도 해당되면 **세션 마지막에 위키를 업데이트**한다:

- 새 기능/페이지/컴포넌트 추가
- 하네스 변경 (rules, commands, hooks, MCP)
- 아키텍처 결정 또는 패턴 변경
- PR/이슈 마무리

단순 버그픽스나 스타일 수정만 있으면 생략해도 된다.

## 위키 경로

```
/tmp/Typing-Room.wiki/
```

## 위키 파일 구조

| 파일                             | 내용                             |
| -------------------------------- | -------------------------------- |
| `Home.md`                        | 프로젝트 개요                    |
| `Claude-Code-하네스-가이드.md`   | 하네스 rules/commands/hooks 전체 |
| `🎨-Typing‐Room-UI-UX-가이드.md` | UI/UX 패턴, 컴포넌트 가이드      |
| `개발-환경-세팅.md`              | 로컬 세팅 가이드                 |

## 업데이트 방법

1. 변경된 내용에 맞는 파일을 찾아 해당 섹션만 수정
2. 새 주제면 적절한 파일에 섹션 추가 (새 파일 생성은 정말 필요할 때만)
3. 변경 후 push:

```bash
cd /tmp/Typing-Room.wiki && git add -A && git commit -m "docs: <변경 내용 요약>" && git push
```

## 작성 스타일

- 한국어로 작성
- 코드 블록은 언어 명시 (`ts`, `bash`, `json` 등)
- 섹션 제목은 `##` / `###` 레벨 사용
- 표로 정리 가능한 건 표 우선
